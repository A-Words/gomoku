# 五子棋对弈游戏 — 设计规格

## 概述

基于 Python + Pygame 开发一款五子棋对弈游戏，支持双人对弈（PvP）和人机对战（PvE），包含三档 AI 难度、悔棋、棋谱保存与回放、经典棋谱学习功能。

## 核心需求

- 15x15 标准棋盘
- 双人对弈 + 人机对战
- AI 三档难度（简单/中等/困难）
- 悔棋功能
- 胜负判定（五子连珠 / 平局）
- 棋谱保存与回放
- 内置经典棋谱学习
- 经典木质视觉风格
- 固定窗口 800x700

## 技术选型

| 项目 | 选择 |
|------|------|
| 语言 | Python 3.10+ |
| 图形库 | Pygame 2.x |
| AI 算法 | Minimax + Alpha-Beta 剪枝 |
| 棋谱格式 | JSON |
| 字体 | 微软雅黑 / 思源黑体 |

## 项目结构

```
WZQ/
├── main.py              # 入口，启动游戏
├── game.py              # 游戏主循环、模式选择、状态管理
├── board.py             # 棋盘逻辑：15x15网格、落子、坐标转换
├── renderer.py          # Pygame 绘制：棋盘、棋子、UI元素
├── ai.py                # AI对手：Minimax + Alpha-Beta，三档难度
├── rules.py             # 胜负判定：五子连珠、平局检测
├── history.py           # 棋谱系统：记录、保存、加载、回放
├── assets/
│   ├── fonts/           # 中文字体
│   └── sounds/          # 落子音效（可选）
├── records/
│   ├── classic/         # 内置经典棋谱
│   └── saved/           # 用户保存的棋谱
└── docs/                # 设计文档
```

### 模块职责

- **game.py** — 调度中心，管理游戏状态（菜单/对弈/回放），协调各模块交互
- **board.py** — 纯逻辑层，15x15 二维数组表示棋盘，提供落子/悔棋/重置接口
- **renderer.py** — 纯绘制层，不包含游戏逻辑，只负责将游戏状态渲染为画面
- **ai.py** — 独立 AI 模块，接收棋盘状态返回落子坐标
- **rules.py** — 判定函数集合，检查五子连珠、平局
- **history.py** — 棋谱的序列化/反序列化（JSON），回放控制

## 游戏流程

### 状态机

```
MENU → PLAYING → GAME_OVER
                ↘ REPLAY
```

状态枚举：
- `MENU` — 主菜单，选择对弈模式和难度
- `PLAYING` — 对弈中
- `GAME_OVER` — 已分胜负，显示结果
- `REPLAY` — 棋谱回放模式

### 对弈主循环

1. 等待当前玩家操作（鼠标点击 或 AI 计算）
2. 校验落子合法性（空位、在棋盘范围内）
3. 更新棋盘状态
4. 记录到棋谱
5. 检查胜负/平局
6. 若无结果 → 切换玩家，回到步骤 1
7. 若有结果 → 进入 GAME_OVER 状态

### 悔棋处理

- PvP 模式：悔一步（移除最后落子）
- PvE 模式：悔两步（玩家 + AI 各一步），保证轮到玩家落子

### 棋盘绘制

- 棋盘区域：560x560 像素，居中偏左
- 格线间距：约 37px（15条线，留边距）
- 星位点：天元 + 四个角星（共5个实心小圆点）
- 坐标标注：A-O 列号 + 1-15 行号

### 棋子绘制

- 黑棋：深色渐变圆 + 左上角高光
- 白棋：浅色渐变圆 + 边缘描边 + 高光
- 最后一手棋：加红色小圆标记

### 视觉风格

- 经典木质风格
- 主色调：木纹黄 #DEB868，深棕线 #5C4A1E
- 字体：微软雅黑或思源黑体
- 按钮：圆角矩形，hover 变色

### 菜单界面

- 标题：五子棋
- 按钮组：双人对弈 / 人机对战 / 棋谱回放
- 人机对战子菜单：选择难度（简单/中等/困难）
- 背景：与棋盘同色系木纹

## AI 对手设计

### 算法：Minimax + Alpha-Beta 剪枝

### 棋型评分表

| 棋型 | 分值 | 说明 |
|------|------|------|
| 连五 | 1000000 | 已形成五子 |
| 活四 | 100000 | 两头空的四 |
| 冲四 | 10000 | 一头被堵的四 |
| 活三 | 5000 | 两头空的三 |
| 眠三 | 500 | 一头被堵的三 |
| 活二 | 200 | 两头空的二 |
| 眠二 | 50 | 一头被堵的二 |

### 评估函数

扫描棋盘四个方向（横、竖、左对角、右对角），统计双方棋型得分：

`局面分 = AI棋型总分 - 玩家棋型总分`

### 难度控制

| 难度 | 搜索深度 | 候选点策略 | 预期表现 |
|------|---------|-----------|---------|
| 简单 | 2层 | 前N选1 + 随机扰动 | 会犯明显错误 |
| 中等 | 4层 | 前N个最优 | 有一定水平 |
| 困难 | 6层 | 全量最优 | 接近人类高手 |

### 优化措施

- **候选点剪枝**：只搜索最后一步棋周围 2 格范围内的空位，大幅减少分支
- **Alpha-Beta 剪枝**：标准优化，减少约一半搜索量
- **评估缓存**：避免重复计算相同局面的评估分

## 胜负判定

### 检测时机

每次落子后立即检测。

### 检测算法

以最后落子为中心，扫描四个方向（横、竖、左对角、右对角），检查是否有连续同色五子。

### 结果类型

- **黑胜**：黑方形成五子连珠
- **白胜**：白方形成五子连珠
- **平局**：棋盘满且无五子连珠（极少见）

## 棋谱系统

### 数据格式

```json
{
  "meta": {
    "date": "2026-05-27",
    "mode": "PvE",
    "difficulty": "中等",
    "result": "黑胜",
    "total_moves": 47
  },
  "moves": [
    {"step": 1, "color": "black", "pos": [7, 7]},
    {"step": 2, "color": "white", "pos": [7, 8]}
  ]
}
```

### 功能清单

| 功能 | 说明 |
|------|------|
| 自动记录 | 对弈过程中每步落子自动记录到内存 |
| 保存棋谱 | 游戏结束或手动保存为 JSON 文件 |
| 加载棋谱 | 从文件加载任意棋谱 |
| 棋谱回放 | 上一步/下一步/自动播放 |
| 接管对弈 | 从回放中途接管，继续对弈 |
| 经典棋谱 | 内置 3-5 局经典五子棋名局 |

### 回放界面

- 底栏显示控制按钮：「◀ 上一步 ▶ 下一步 ⏯ 自动播放」
- 显示当前手数 / 总手数
- 「接管」按钮可从当前局面接管为对弈

### 文件存储

- 内置棋谱：`records/classic/` 目录
- 用户棋谱：`records/saved/` 目录
- 文件名格式：`YYYY-MM-DD_HH-MM_mode.json`
